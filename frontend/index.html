CSS 变量参考
所有样式通过 CSS 自定义属性统一管理，禁止在组件中硬编码颜色值。
核心变量

--s1/s2/s3表面层级，越大越亮
--card卡片专用背景色
--text/2/3文字三级层次
--accent主强调色（紫）
--accent-s主色半透明背景
--accent-g主色较深透明边框
--teal成功色（青绿）

--red危险 / 错误色
--border/2分隔线 / 控件边框
主题切换模式
:root { /* 深色默认 */ --accent: #8B72FF; --bg: #07070E; } @media(prefers-color-scheme:light) { :root { --accent: #6B52F0; --bg: #F2F2F7; } }
· 颜色必须使用变量，禁止硬编码
· 背景层级用 --s1/s2/s3，不用具体色值
· 阴影颜色基于 --accent 而非纯黑
· 动效缓动统一用 cubic-bezier(.32,.72,0,1)
iframe 运行规范
所有 App 运行在父页面的 iframe 内，部分 DOM/BOM API 会穿透 iframe 边界，必须遵守以下规则。
禁止使用
· element.scrollIntoView() — 会触发父页面滚动，造成意外导航
· window.parent.* — 禁止访问父框架对象
· window.top.* — 禁止访问顶层框架
· window.location = ... — 禁止强制跳转 URL
替代方案
// ❌ 错误：scrollIntoView 会穿透 iframe element.scrollIntoView({ behavior: 'smooth' }); // ✅ 正确：只在容器内部滚动 const container = document.getElementById('my-scroll-area'); container.scrollTop = container.scrollHeight;
滚动容器原则
· 每个 App 必须有明确的滚动容器（设 overflow-y:auto）
· 滚动操作只操作该容器的 scrollTop，不调用全局 API
· body 高度应固定为 100vh，禁止让 body 本身可滚动
API 调用规范
· 所有 API 请求用相对路径 /api/...，由 Caddy 反代
· 禁止硬编码 IP 或端口
· 错误必须在 App 内部展示，禁止 alert()
